Problem

Today you either trust the agent too much or babysit every move.

Most agent products stop at “connect wallet”. That is not enough for real money. Treasury owners need a clean way to say: the agent can use this budget, but not the rest.

Product

A budget guard that sits in front of every agent action.

Yield Charter checks budget, route, protocol, counterparty, and risk before execution. The agent does not spend first and explain later.

Who It Is For

Teams running trading, rebalancing, or ops agents with real funds.

If your agent needs to move money on X Layer, this product gives you a usable operating model: small budget, hard rules, and visible receipts.

How To Use

Try the app in 30 seconds

This app is for a treasury owner. You do not hand the whole wallet to the agent. You release a smaller budget, let the agent ask to act, and the product enforces the money boundary for you.

1. Click Set Budget to release an operating budget while keeping principal locked.
2. Click Run Agent Request to see how the guard resizes or approves spending.
3. Click Pause Agent to watch the same request get blocked before execution.
4. Click Reset Live State to go back to the current live packet.
  • This is not a generic wallet dashboard. It is a spending-control product for autonomous agents.
  • The core promise is simple: the agent can spend released yield, but not treasury principal.
  • Every allowed spend comes back with a receipt, a policy rationale, and tx proof.
